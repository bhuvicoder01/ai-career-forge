3. ABSTRACT
================================================================================

AI CareerForge is a production-grade, full-stack career management platform that
uses Agentic Retrieval-Augmented Generation (RAG) and Semantic Vector Search to
automate the entire job application lifecycle. The platform enables users to
upload their resume, have it parsed by Google Gemini AI, discover semantically
matched job opportunities from global APIs, and generate fully tailored
application materials — resumes, cover letters, email introductions, and
interview preparation kits — with a single click.

The system is built on a modern microservice-oriented architecture using
Spring Boot 3.3 (Java 21) for the backend and Next.js 15 (App Router) for the
frontend. It employs MongoDB Atlas Vector Search with 768-dimensional embeddings
for semantic job matching, AWS S3 for document storage, and Server-Sent Events
(SSE) for real-time background processing feedback.

4. INTRODUCTION
================================================================================

In today's hyper-competitive job market, candidates face "application fatigue" —
the exhausting cycle of manually searching for jobs, tailoring resumes for each
posting, and preparing for interviews. Existing platforms like LinkedIn and
Indeed provide vast databases but rely on keyword-based matching, which misses
semantic relevance. Resume builders like Canva and Zety offer templates but
cannot tailor content to specific job descriptions. Generic AI tools like
ChatGPT require complex prompting and are not integrated with job data.

AI CareerForge bridges all these gaps by providing a unified, intelligent
platform that:

  - Parses uploaded resumes using Google Gemini AI to extract structured
    profile data (skills, experiences, internships, certifications, goals).
  - Fetches real-time job listings from the Adzuna API across global markets.
  - Performs semantic vector search using MongoDB Atlas to find jobs that
    are genuinely relevant to the user's profile — not just keyword matches.
  - Generates complete, tailored application kits (resume PDF, cover letter,
    email introduction, interview prep) using multi-agent AI orchestration.
  - Tracks all applications in a mission-control-style dashboard with
    status management, archiving, and deletion capabilities.
  - Provides a floating AI assistant for real-time career Q&A.

5. PROBLEM STATEMENT
================================================================================

The modern job seeker faces several critical bottlenecks:

1. SEARCH INEFFICIENCY: Sifting through thousands of irrelevant jobs using
   outdated keyword filters that miss semantically relevant positions.

2. LACK OF PERSONALIZATION: Using a "one-size-fits-all" resume leads to lower
   ATS (Applicant Tracking System) scores and reduced callback rates.

3. APPLICATION FATIGUE: Writing unique cover letters and email introductions
   for every application is extremely time-consuming.

4. INTERVIEW ANXIETY: Lack of role-specific, company-specific interview
   preparation materials tailored to the actual job description.

5. FRAGMENTED WORKFLOW: Using 5+ different tools (job boards, resume builders,
   AI writers, email clients, note apps) creates friction and context-switching.

  [Next.js 15 Frontend]  <--->  [Spring Boot 3.3 REST API]  <--->  [MongoDB Atlas]

        |                              |           |                 [Vector Store]

        |                              |
        +--- SSE Stream ---- [Background Sync Service]

   [Zustand Global State]

8. DETAILED MODULE DESCRIPTIONS
================================================================================

8.1 AUTHENTICATION MODULE
  Files: AuthController, AuthService, JwtService, JwtAuthenticationFilter,
         OAuth2SuccessHandler, SecurityConfig
  
  - JWT-based stateless authentication for email/password login.
  - OAuth 2.0 with Google for one-click social login.
  - BCrypt password encoding for secure credential storage.
  - Global CORS configuration for cross-origin frontend requests.
  - Token expiration detection on app startup for automatic logout.

8.2 USER PROFILE MODULE
  Files: UserProfileController, UserProfileService, ProfileAiAgent
  
  - Resume Upload: Users upload a PDF resume which is stored in AWS S3.
  - AI Resume Parsing: PDFBox extracts raw text, then Google Gemini AI
    parses it into structured fields:
      * Skills (List<String>)
      * Work Experiences (title, company, duration, description)
      * Internships (role, company, duration, description)
      * Academic Projects (title, technologies, description)
      * Certifications (List<String>)
      * Parsed Career Goals
  - Profile Editing: Users can manually update skills, preferred location,
    salary expectations, and lifestyle preferences.
  - On every profile update or resume upload, old job data is purged and
    a background job sync is automatically triggered.

8.3 JOB DISCOVERY & RECOMMENDATION MODULE
  Files: JobController, JobService, JobRecommendationAgent, AdzunaClient,
         AdzunaConfig, GeminiEmbeddingModel

  - Adzuna API Integration: Fetches real-time job listings from global
    markets with configurable country codes and API credentials.
  - Vector Indexing: Every fetched job is converted to a 768-dimensional
    embedding vector using Google's text-embedding-004 model and stored
    in MongoDB Atlas Vector Search.
  - Semantic Search: When a user requests recommendations, their full
    resume text is embedded and a cosine similarity search is run against
    all indexed jobs.
  - Multi-Factor Match Scoring (65-100 range):
      * Base Score: 70 points
      * Vector Similarity Bonus: 0-15 points (from embedding distance)
      * Skill Overlap Bonus: 0-10 points (+2 per matched skill, capped)
      * Experience Alignment Bonus: 0-5 points (parses JD year requirements
        and compares to user's actual experience in years)
  - AI Relevance Explanations: Google Gemini generates a personalized
    explanation for why each job matches the user, cached in the database
    for instant subsequent loads.
  - Company Culture Analysis: AI-powered analysis of company culture and
    work environment rendered as Markdown.
  - Fair Pay Intelligence: AI-generated salary benchmarking.
  - Jobs are sorted in descending order by match score.

8.4 BACKGROUND JOB SYNCHRONIZATION MODULE
  Files: JobSyncService, SyncSseEmitterRegistry, JobSyncStatus,
         JobSyncStatusRepository

  - Asynchronous Processing: Uses Spring @Async to run job syncing in a
    background thread, completely independent of HTTP connections.
  - Trigger: Automatically triggered after resume upload or profile update.
  - Process: Iterates through the user's top 3 skills, calls the Adzuna API
    for each, and indexes results into the vector store.
  - Real-Time Status: Uses Server-Sent Events (SSE) to push sync progress
    to the frontend in real-time (e.g., "Syncing Java roles... 2/3").
  - Persistent Status Tracking: Sync status is persisted in MongoDB so that
    if the user navigates away and returns, the current state is immediately
    available.
  - Stale Recovery: If the server crashes mid-sync, the system detects
    stale SYNCING records (>5 minutes without update) and auto-resets them
    to IDLE on the next status check.
  - SSE Emitter Registry: Manages per-user SSE connections with automatic
    cleanup on timeout, error, and completion.
  - Global Frontend State: The SSE connection is managed at the dashboard
    layout level using a Zustand store, so it survives page navigation.
    A sync progress banner is visible on ALL dashboard pages.

8.5 APPLICATION TRACKER MODULE
  Files: ApplicationController, ApplicationTrackerService, ApplicationPrepAgent,
         PdfGenerationService, Application (model)

  - One-Click Tailoring: From any job detail page, users can trigger a
    complete application generation pipeline with a single button click.
  - AI Agent Pipeline (multi-step):
      Step 1: Resume Tailoring — Gemini AI analyzes the JD and user profile
              to generate optimized resume content (summary, experiences,
              projects, skills, certifications, internships).
      Step 2: Communication Kit — Gemini generates a personalized cover
              letter and professional email introduction.
      Step 3: Interview Prep Kit — Gemini generates a comprehensive
              interview preparation document with role-specific questions,
              company-specific insights, and behavioral prompts.
  - PDF Resume Generation: Tailored resume content is rendered into a
    professional PDF using Thymeleaf HTML templates and OpenHTMLtoPDF:
      * MODERN template: Two-column layout with colored sidebar.
      * CLASSIC template: Traditional single-column, serif-based layout.
    Both templates use safe bracket-access patterns to handle variable
    data types from the AI (Map vs String).
  - S3 Upload: Generated PDFs are uploaded to AWS S3 with presigned URLs
    for secure, temporary access.
  - Status Management: Applications can be tracked through statuses:
    SAVED → APPLIED → INTERVIEW → OFFER / REJECTED / ARCHIVED.
  - Ownership Security: Backend enforces strict ownership checks — users
    can only view, modify, or delete their own applications.
  - Prep Materials Page: Tabbed view showing tailored resume (with
    side-by-side comparison to original), cover letter, email intro,
    and interview prep kit — all with mobile-responsive layouts.

8.6 FLOATING AI ASSISTANT MODULE
  Files: ChatController, ChatService, FloatingAiAssistant (component)

  - A floating bubble on every dashboard page that opens a chat interface.
  - Powered by Google Gemini for real-time career advice Q&A.
  - Context-aware: Can answer questions about the user's profile, job
    matches, and preparation strategies.

8.7 COMPANY INTELLIGENCE MODULE
  Files: CompanyIntelligenceService

  - Uses Gemini AI to generate company culture and environment analysis.
  - Displayed on job detail pages as formatted Markdown content.
  - Provides insights about work-life balance, team dynamics, and
    company values based on publicly available data.

12. KEY ALGORITHMS & TECHNIQUES
================================================================================

12.1 SEMANTIC VECTOR SEARCH
  - User's resume text is converted to a 768-dimensional vector using
    Google's text-embedding-004 model.
  - Job descriptions are similarly embedded and stored in MongoDB Atlas
    Vector Search with a cosine similarity index.
  - At query time, the user vector is compared against all job vectors
    to find semantically similar jobs (threshold: 0.1).

12.2 MULTI-FACTOR MATCH SCORING (65-100 RANGE)
  - Base Score: 70 points.
  - Factor 1 — Vector Similarity: 0-15 bonus points based on cosine
    distance from embedding search.
  - Factor 2 — Skill Overlap: 0-10 bonus points (+2 per user skill
    found in the job description, capped at 10).
  - Factor 3 — Experience Alignment: 0-5 bonus points.
    The algorithm parses the user's experience durations (handles formats
    like "2020-2023", "3 years", "Jan 2019 - Present", "6 months") and
    compares total years against the JD's requirements (e.g., "5+ years").
    * Meets/exceeds requirement: +5 points
    * Within 1 year: +3 points
    * Below requirement: +1 point
    * No explicit requirement in JD: +2.5 points if user has experience.
  - All scores clamped to [65, 100] range.

12.3 AGENTIC RAG FOR DOCUMENT GENERATION
  - Multi-step AI pipeline using Google Gemini:
    * Agent 1 (Resume Tailor): Analyzes JD + user profile to generate
      optimized, structured resume JSON data.
    * Agent 2 (Communication Kit): Generates cover letter + email intro.
    * Agent 3 (Interview Prep): Generates comprehensive prep kit with
      role-specific questions and company insights.
  - Rate limiting between agent calls to respect API quotas (2s sleep).
  - TypeReference-based JSON deserialization for type-safe data handling.

12.4 BACKGROUND SYNC WITH SSE
  - Spring @Async for non-blocking background processing.
  - SseEmitter-based real-time event streaming per user.
  - Auto-reconnecting SSE client using fetch() + ReadableStream API
    (supports JWT auth headers, unlike native EventSource).
  - Zustand global state store at the dashboard layout level ensures
    sync status persists across page navigation.
  - Stale sync detection (>5 min without update = auto-reset to IDLE).
